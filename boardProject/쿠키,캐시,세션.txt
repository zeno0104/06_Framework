쿠키 (Cookie) : 
클라이언트(브라우저)에 저장되는 작은 데이터 조각.
클라이언트가 서버에 요청을 보낼때 쿠키를 포함하여 보낼 수 있음.
주로 사용자 인증, 설정, 장바구니 항목 등을 저장하는데 쓰임
쿠키는 클라이언트측에서 쉽게 수정할 수 있기때문에 보안에 취약할 수 있음
만료날짜를 지정하여 생명주기를 설정할 수 있으며, 만료 날짜를 지정하지 않으면
브라우저가 닫힐때까지 유지.

캐시 (Cache) : 
클라이언트에(브라우저)저장하는 방식.
자주 사용하는 데이터를 임시로 저장해두는 기술.
이미지, css, js 같은 정적 파일을 기억함. 
브라우저가 한번 받았던 서버의 응답파일(js/css)을 저장해두고, 
같은 요청이 오면 서버에 재요청하는것이 아니라 캐시에 저장한걸 꺼내서 씀

세션 (Session) : 
서버측에서 사용자의 상태를 관리하는데 사용됨.
클라이언트가 서버에 처음 접속하면 서버는 해당 클라이언트에 대한 고유한 세션 ID 생성, 저장함.
클라이언트는 세션 ID를 서버에게 전달받아 쿠키에 저장해둠.
이후 해당 클라이언트의 요청이 있을때마다 세션 ID가 사용되어 세션에 저장된 정보를 이용함.
-> 클라이언트의 요청 시 쿠키가 함께 서버로 전송됨
세션은 보안상 쿠키보다 안전하며, 
서버측에서 데이터를 관리하기 때문에 클라이언트 측에서 수정 불가.
세션은 사용자가 로그아웃, 세션이 만료되거나, 브라우저가 전부 꺼질때까지 지속됨.
